Last update: 16th February 2017

GE Power provides two mapping services on Predix:

  • Intelligent Mapping — stores the location and properties of static assets
  • Dynamic Mapping — stores the current and historical locations of moving assets

These mapping services provide APIs to facilitate application development. Using these mapping APIs, you can develop a range of applications with geospatial context, helping you locate and visualise your data to support analysis, planning and decision making.

This documentation provides:

  • Instructions on how to get started with Intelligent Mapping and Dynamic Mapping services
  • Examples of how to bind the services to OpenLayers 3 (OL3) map for displaying data on a two-dimensional map
  • Examples of how to bind the services to a Cesium map for displaying data on a three-dimensional map
  • Access to our Intelligent Map Designer application, that provides a simple interface to load and validate your data
  • A simple Polymer map that is compatible with other PX user interface components to display assets on a map

Disclaimers:

  • Data is not encrypted and all text is displayed in human readable format. Personally identifiable information should not be stored in the system.
  • Data retention in Dynamic Mapping can be controlled via the APIs and we request that you limit historical locations to a maximum of 3 months. If users of the service do not adhere to this policy, we may periodically remove any data from the Dynamic Mapping service that is more than 3 months old.
