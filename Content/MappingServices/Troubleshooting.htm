This article provides troubleshooting for some of the errors you may encounter during Predix development.

401 Errors

If you get a 401 error code when sending requests to the Intelligent Mapping or Dynamic Mapping services, check that:

  • The following headers are present:
    • Authorization
    • Predix-Zone-Id
    • X-Subtenant-Id
  • The specified Predix-Zone-Id has got the scope and authorization for the services you are trying to use , for example intelligent-mapping.zones.<predix-zone-id>.user
  • The UAA you are using as the trustedIssuerIds has been specified correctly when the service instance was created, that is -c'{"trustedIssuerIds":["https://…../oauth/token"]}'

400 Errors

If you get a 400 error code when posting GeoJSON to Intelligent Mapping, check that the GeoJSON is valid.
