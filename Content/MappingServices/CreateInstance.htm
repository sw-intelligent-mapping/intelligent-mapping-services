Before you begin

A UAA service instance must be already configured as your trusted issuer.

Creating a mapping service instance using the Cloud Foundry CLI

  1. List the services in the Cloud Foundry marketplace
  2. cf marketplace
  3. The Intelligent Mapping service is intelligent-mapping and the Dynamic Mapping service is dynamic-mapping.

  4. This example uses intelligent-mapping, but the steps are the same for dynamic-mapping.

  5. Create a service instance using one of the following:
    • UNIX:
    • cf create-service intelligent-mapping <plan> <my_service_instance_name> -c '{"trustedIssuerIds":["<my_uaa_instance_issuerId>", "<my_uaa_instance2_issuerId>"]}'
    • Windows:
    • cf create-service intelligent-mapping <plan> <my_service_instance_name> -c "{\"trustedIssuerIds\":[\"<my_uaa_instance_issuerId>\", \"<my_uaa_instance2_issuerId>\"]}"
  6. Where:
  7. <plan> The plan associated with a service. In this case, use Tiered.
    <my_service _instance_name> The name you are planning to give to this service instance.
    <my_uaa_instance_issuerId> The issuerID (base URL) of your trusted issuer (UAA instance). For example, https://13fa0384-9e2a-48e2-9d06-2c95a1f4f5ea.<productionurl>/oauth/token. After you have bound the UAA instance to your application, you can retrieve the URI from the VCAP_SERVICES environment variable. If applicable, you can use a comma-delimited format to specify multiple trusted issuers (UAA instances), such as ["<my_uaa_instance1_issuerId>", "<my_uaa_instance2_issuerId>"].
  8. UNIX example:
  9. cf create-service intelligent-mapping Tiered example-ims -c '{"trustedIssuerIds":["https://13fa0384-9e2a-48e2-9d06-2c95a1f4f5ea.predix-uaa.run.aws-usw02-pr.ice.predix.io/oauth/token", "https://7c6f4184-6baa-4946-8d44-06c16b6c7646.predix-uaa.run.aws-usw02-pr.ice.predix.io/oauth/token"]}'
  10. Windows example:
  11. cf create-service intelligent-mapping Tiered example-ims –c "{\"trustedIssuerIds\":[\"https://13fa0384-9e2a-48e2-9d06-2c95a1f4f5ea.predix-uaa.run.aws-usw02-pr.ice.predix.io/oauth/token\", \"https://7c6f4184-6baa-4946-8d44-06c16b6c7646.predix-uaa.run.aws-usw02-pr.ice.predix.io/oauth/token\"]}"

  12. Trusted issuer IDs may vary dependant on Predix.io registration. In the above examples, the UAA was created with a Predix.io registration on Predix US West.

  13. Cloud Foundry displays a message confirming that the service instance has been created.

Creating a mapping service instance using a web browser

  1. Sign in to your Predix cloud account at Predix.io.
  2. From the Predix catalog, choose either Intelligent Mapping or Dynamic Mapping.
  3. Click on the Subscribe button for the tiered plan.
    • Select the Org and Space for your UAA
    • Select the UAA
    • Fill out the Service instance name field with name you are planning to give to this service instance
    • Enter Service plan of Tiered
  4. Click Create Service.
  5. A new page will provide details of the newly created service.
