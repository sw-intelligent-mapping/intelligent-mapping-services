Adding Intelligent Mapping or Dynamic Mapping services to your application

Send feedback

Last update: 28th September 2016

Before you begin

You must update your OAuth2 client to use your service instance before you perform this task.

Procedure

This process is in two parts:

  • Getting a bearer token from UAA, which is used in the second part.
  • Making sure client applications can access geospatial data using Intelligent Mapping or Dynamic Mapping service REST API endpoints.
Getting a Bearer Token from UAA

In a REST client, Authorization tab:

  1. Choose the GET method.
  2. Enter the URI of your UAA instance, such as:
  3. https://<your_uaa_instance_uri>/oauth/token?grant_type=client_credentials

  4. Make sure you include grant_type=client_credentials in your URI.

  5. Add the following information as shown in the graphic below the table.
  6. Type Select Basic Auth from the drop-down list.
    Username Enter the client ID that you created in Updating OAuth2 client to use your service instance.
    Password Enter the client secret that you created in Updating OAuth2 client to use your service instance.
  7. Click Send.
  8. Select the access_token that is returned and copy it to your clipboard. This is the token that you need in the Ensuring client apps can access Intelligent mapping data procedure.
  9. Verify that the scope given is the same as used in the VCAP_SERVICES environment variable
Ensuring client apps can access Intelligent mapping data

In a REST client:

  1. Choose the POST method.
  2. Enter the URI of your service instance, from the VCAP_SERVICES environment variable. To post data, enter:
  3. https://<production URL>/v1/collections/coll1
  4. Add the following HTTP headers to the request:
  5. Content-Type Enter the value application/json.
    Authorization Enter the token you receive from UAA in the form of Bearer <token from trusted issuer> in the Getting a Bearer Token from UAA procedure.
    Predix-Zone-Id Enter the ID required to make REST calls, using the instanceId value specified in the VCAP-SERVICES environment variable.
    X-Subtenant-Id

    Enter an ID to identify a subtenant within the Predix-Zone-Id. Subtenants can be used to allow different access privileges for collections. The subtenant IDs must match the required collection.

    If no subtenants are needed, use the Predix-Zone-Id.
  6. Enter a well-formed GeoJSON string in the body of your request to store the GeoJSON data in a new collection called coll1.
