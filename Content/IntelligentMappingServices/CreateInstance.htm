Before you begin

A UAA service instance must be already configured as your trusted issuer.

The following steps are performed using the Cloud Foundry CLI. To complete the steps in a web browser, follow the instructions on the service page in the Predix Catalog.

Procedure

  1. List the services in the Cloud Foundry marketplace
  2. cf marketplace
  3. The Intelligent Mapping service is intelligent-mapping and the Dynamic Mapping service is dynamic-mapping.

  4. This example uses intelligent-mapping, but the steps are the same for dynamic-mapping.

  5. Create a service instance
  6. cf create-service intelligent-mapping <plan> <my_service_instance_name> -c '{"trustedIssuerIds":["<my_uaa_instance_issuerId>", "<my_uaa_instance2_issuerId>"]}'
  7. Where:
  8. <plan> The plan associated with a service. In this case, use Beta.
    <my_service _instance_name> The name you are planning to give to this service instance.
    <my_uaa_instance_issuerId> The issuerID (base URL) of your trusted issuer (UAA instance). For example, https://13fa0384-9e2a-48e2-9d06-2c95a1f4f5ea.<productionurl>/oauth/token. After you have bound the UAA instance to your application, you can retrieve the URI from the VCAP_SERVICES environment variable. If applicable, you can use a comma-delimited format to specify multiple trusted issuers (UAA instances), such as ["<my_uaa_instance1_issuerId>", "<my_uaa_instance2_issuerId>"].
  9. Cloud Foundry displays a message confirming that the service instance has been created.

Cloud Foundry CLI syntax can differ between the Windows and Linux operating systems. See the Cloud Foundry help for the correct syntax for your OS.
